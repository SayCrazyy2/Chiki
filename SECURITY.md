# Security Policy

## Reporting a Vulnerability
Please do not open public issues for security vulnerabilities.

Email maintainers directly with:
- Impact summary
- Reproduction steps
- Suggested remediation

We will acknowledge reports promptly and provide remediation updates.

## Security Model (Milestone 1)
- Owner-only Telegram access
- Explicit approval flow for high-risk tools
- Structured audit logging for tool actions and approval decisions
- Path traversal protections for file tool
- Terminal allowlist/denylist and timeouts
